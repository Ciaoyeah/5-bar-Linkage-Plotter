Webots

1. 什麼是 Webots？

Webots 是由 Cyberbotics 公司開發，並於 2018 年正式開源的機器人模擬器。它支援多種作業系統（Windows, Linux, macOS），廣泛應用於工業機器人、自動駕駛、無人機以及多 agent 系統的開發與教學。

其核心優勢在於能夠提供精確的物理運算與擬真的視覺傳饋，讓使用者在將演算法部署到實體硬體前，先在虛擬環境中完成驗證。

2. 核心技術特點

  • 強大的物理引擎 (ODE)： 採用 Open Dynamics Engine (ODE)，能夠精確模擬剛體動力學、碰撞檢測與摩擦力，非常適合測試如機械手臂的運動軌跡與受力分析。

  • 豐富的感知系統： 內建多種感測器模型，包括：

    • 雷達 (Lidar)、相機 (Camera)、超音波、GPS。

    • 力感測器 (Force Sensors)、慣性測量單元 (IMU)。

  • 跨語言開發環境： 支援 Python、C/C++、Java、MATLAB 及 ROS/ROS2。對於偏好 AI 與 OpenCV 整合開發的使用者，Python API 提供了極佳的靈活性。

  • 高相容性 CAD 導入： 支援從 Fusion 360、SolidWorks 導入 URDF 或 PROTO 模型，方便將自行設計的機械零件直接轉化為模擬物件。

Webots 核心組件解析：Solid 與 HingeJoint

在 Webots 的世界中，建構一個機器人就像是在搭樂高。要讓機器人能看、能動、能受重力影響，必須深入理解 Solid 與 Joint 節點的運作邏輯。

1. Solid 節點：賦予物體物理靈魂

Solid 是 Webots 中所有具備「物理屬性」物件的基石。如果你希望一個零件擁有質量、會碰撞、或能被感測器偵測，它必須是一個 Solid。

關鍵欄位解析

  • Translation / Rotation：定義物件在 3D 空間中的初始位置與姿態。

  • Children (子節點)：通常包含 Shape（視覺外觀），定義物件看起來長什麼樣子。

  • BoundingObject (碰撞邊界)：這決定了物件的「實體形狀」。若未設定，物件會像幽靈一樣穿過地面或其他零件。通常使用簡單幾何體（如 Box 或 Cylinder）來優化運算速度。

  • Physics (物理屬性)：這是最關鍵的一環。

    • 若此欄位為空，物件將固定在空中（如桌子、地板）。

    • 若賦予 Physics 節點，則可設定 質量 (Mass)、中心點 (Center of Mass) 與 慣性張量，使物件受到重力與外力影響。

2. HingeJoint 節點：實現旋轉自由度

HingeJoint（旋轉關節）提供了一個自由度（1-DOF），模擬像門鉸鏈或機械手臂關節的旋轉運動。它定義了兩個 Solid 之間的相對運動關係。

HingeJoint 的三大組成部分

  1. jointParameters (關節參數)：

    • Anchor (錨點)：定義旋轉中心的座標（例如手臂關節的中心）。

    • Axis (軸線)：定義繞著哪個向量旋轉（例如 $z$ 軸 $[0, 0, 1]$）。

    • Damping Constant (阻尼係數)：模擬物理摩擦力，防止關節過度晃動。

  2. device (設備層)：

    • 這裡是你放置 RotationalMotor（馬達）、PositionSensor（位置感測器）或 Brake（煞車）的地方。這讓你可以透過 Python 程式碼控制關節的角度或轉速。

  3. endPoint (端點)：

    • 這是「關節另一頭連接的物體」。通常是另一個 Solid 節點。這形成了 Webots 的樹狀結構。
